# 3. 游戏设计

## 玩家跳跃能力的等级：

- 跳跃,单方向。
- 跳跃键按下时长会决定跳跃的最高点。
- 能够达到二段跳的高度，加上左右移动。
- 在空中可以通过左右移动进行细微的调整。
- 三段跳的高度，最高高度。
- 攀爬技能。
- 第三次跳跃的时候反向跳跃。
- 冲刺的使用。
- 火箭。

## 技巧

- 按下之后不松手，到达跳跃的最高点之后再马上按第二次跳跃如此往复。
- 跳跃之后，可以在空中去控制，左右。
- 第三次跳跃的时候反向跳跃

### 开始剧情

这是一个关于友情的故事。

凉鞋有一个朋友 A，

梦想是要去天空寻找传说中的宝藏。

但是朋友 A 在一次意外中去世了。

在临死前凉鞋答应 A 替 A 完成梦想，

在 A 的葬礼之后，凉鞋开始履行承诺。

踏上了上天之路。



### 通关剧情

凉鞋终于找到了传说中的宝藏，实现了 A 的梦想。（A 托梦给了主角）



## Idea/想法

- 时间越少越厉害
- 目的：为了让玩家看到地图全貌
  - 小地图
  - 一个快捷键可以让摄像机距离边远。
- 角色和关卡物体都是用的插件的。
- 学习美术插件。

## 反馈/建议

* 冲刺 和 火箭  可能需要在通关到指定关卡之前 功能给禁掉，如果之前 就知道了这个功能 可能会就一直用这个功能，那么前几关 就没啥意义了。
* 这种跳台，可能需要对应 每一种场景 更换颜色 或者造型。 
* 比如 雪场景 换成 雪梯  ，  水场景 换成木头 巴拉巴拉的
* 像素颗粒度大小 可能需要调整一下。



# 4. 游戏制作方法论


## 做游戏的时候

切换思维

* 技术思维
* 设计角度（策划思维）
* 美术/音乐角度

## 一个游戏的结构

* 关卡有结束
* 惩罚
  * 角色死亡
  * 血量
  * 重玩
  * 生命数量减掉 1 
* 成就感


## 游戏制作流程

1. 分析，定目标，确定三个关卡。
2. 第一章：可玩的版本 MVP(最小可验证版本)
3. 第二章: 扩充到了是个关卡，涉及一点美术的东西。



## 游戏制作的方法论

1. 先有一个可玩的版本，比较模糊的目标。
2. 不断玩。
3. 玩的过程中完善。
4. 重复 2 ~ 4。
